# speckit-autopilot: bootstrap-product

Read `docs/product.md` in the current working directory and produce:
- `docs/roadmap.md` – ordered list of epics and features with dependencies
- `docs/product-backlog.yaml` – machine-readable backlog with feature metadata
- `docs/autopilot-state.json` – initial autopilot state

**Do NOT write any application code.** This skill only produces planning artifacts.

## Steps

1. Read `docs/product.md`. If it does not exist, stop and ask the user to create it.
2. Use the `product-planner` agent to analyse the document and extract:
   - Epics and features
   - Dependencies between features
   - Suggested implementation order
   - Acceptance criteria per feature
3. Write `docs/roadmap.md` using the template below.
4. Write `docs/product-backlog.yaml` using the schema below.
5. Write `docs/autopilot-state.json` with mode=greenfield and status=bootstrapped.
6. Print a summary table of the backlog.

## roadmap.md template

```markdown
# Product Roadmap

Generated by speckit-autopilot on {ISO_DATE}

## Epics

{EPIC_LIST}

## Feature Order

| # | Feature | Epic | Status | Depends On |
|---|---------|------|--------|------------|
{FEATURE_ROWS}

## Notes

{NOTES}
```

## product-backlog.yaml schema

```yaml
version: "1"
generatedAt: "{ISO_DATE}"
features:
  - id: "F-001"
    title: "Feature title"
    epic: "Epic name"
    status: open          # open | in_progress | done | blocked
    priority: high        # high | medium | low
    dependsOn: []         # list of feature IDs
    acceptanceCriteria:
      - "Criterion 1"
      - "Criterion 2"
    estimatedComplexity: medium   # low | medium | high
    specKitBranch: ""     # filled when work starts
    notes: ""
```

## Spec Kit detection

Before writing any file, check whether Spec Kit is available:
- Run `specify --version` or check for `.speckit/` directory
- If Spec Kit is NOT installed/initialised: print a notice and suggest running
  `specify init . --ai claude --ai-skills` before using `ship-product` or `ship-feature`
- Bootstrap proceeds regardless; Spec Kit is only needed at implementation time
